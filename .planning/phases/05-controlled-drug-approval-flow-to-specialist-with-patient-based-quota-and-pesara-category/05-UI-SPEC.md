## Interaction Contract

### 1. DoctorRequest form — Pesara checkbox

**Location:** Below the `no_ic` (IC number) field, before the drug selector.

**Behaviour:**
- Renders as `<Checkbox id="is_pesara">` with a `<Label htmlFor="is_pesara">` wrapping "Pesara (Government Retiree)"
- Default: unchecked
- When checked: no visual change to the rest of the form — the Pesara flag is purely informational at submission time
- Submitted value: `is_pesara: boolean` added to the `dispensing_requests` insert payload

**Validation:** No validation required — the checkbox is optional and defaults to false.

### 2. SpecialistDashboard — Controlled Drug tab sub-tabs

**Structure:** The existing "Controlled Drug" `TabsContent` panel gains a nested `Tabs` component with two `TabsTrigger` labels:
- `"Regular"` — with a pending count Badge using the existing sidebar-badge pattern: `bg-amber-500 text-white rounded-full text-xs px-1.5 ml-1`
- `"Pesara"` — same pending count Badge pattern

**Default active tab:** `"Regular"` (shows first, most common workflow)

**Regular sub-tab — request row layout:**
- Each row in the pending table shows: Drug name | Patient name | IC | Qty | Requested (time ago) | **Quota Badge** | Actions
- Quota Badge occupies its own table cell, right-aligned, using the semantic badge classes defined above
- Quota Badge format: `"{used}/{limit} patients"` — e.g. `"45/60 patients"`
- When `used >= limit`: badge class `bg-red-100 text-red-700 border-red-300` + label `"Quota Exhausted: {used}/{limit}"`
- When `used >= limit * 0.8`: badge class `bg-amber-100 text-amber-700 border-amber-300` + label `"{used}/{limit} patients"`
- When `used < limit * 0.8`: badge class `bg-green-100 text-green-700 border-green-300` + label `"{used}/{limit} patients"`
- When no quota set for drug: badge class `bg-gray-100 text-gray-600 border-gray-300` + label `"No quota set"`

**Pesara sub-tab — request row layout:**
- Same columns as Regular minus the Quota Badge column
- Replace quota cell with static `<Badge>` using `bg-blue-100 text-blue-700 border-blue-300` and label `"Unlimited"`

### 3. SpecialistDashboard — Approve dialog (Controlled Drug, Regular sub-tab)

**Normal state (quota not exhausted — used < limit):**
- Existing approve dialog structure: notes Textarea + Approve / Cancel buttons
- No borrow-from-facility field shown

**Quota exhausted state (used >= limit):**
- At top of dialog body, before the notes Textarea, render:
  ```
  <Alert variant="destructive">
    <AlertDescription>
      Quota exhausted: {used}/{limit} patients for {drug_name} this year.
      Approval will exceed the annual patient quota.
    </AlertDescription>
  </Alert>
  ```
- Below the Alert, add:
  ```
  Label: "Borrowing quota from facility"
  Input: placeholder="Enter facility name" (free-text, required when shown)
  ```
- The borrow field is conditionally rendered only when `used_count >= quota_limit`
- If borrow field is visible, it is required — form submit is disabled until filled
- Submitted value: `borrowed_from_facility: string` added to the approve mutation payload

**Approve dialog — Pesara sub-tab:**
- Standard notes Textarea + Approve / Cancel only — no quota alert, no borrow field

### 4. FmsDashboard — Quota table new column

**Table enhancement:** Add a "Pesara" column after the existing "Patients Used / Limit" column.

**Column header:** `"Pesara"`

**Cell content:** `"{pesara_count} (Unlimited)"` — e.g. `"12 (Unlimited)"`
- Text: 14px, weight 400, `text-muted-foreground` color when pesara_count is 0
- Text: 14px, weight 600, `text-foreground` color when pesara_count > 0

**Layout choice (Claude's discretion applied):** Separate column — cleaner than a secondary line inside the existing cell, avoids line-height overflow in existing rows.

### 5. Loading and empty states

| Context | State | Treatment |
|---------|-------|-----------|
| SpecialistDashboard sub-tabs loading | Loading | `<Skeleton>` rows — same pattern as existing pages |
| Regular sub-tab — no pending requests | Empty | Render message inside TabsContent (see Copywriting Contract) |
| Pesara sub-tab — no pending requests | Empty | Render message inside TabsContent (see Copywriting Contract) |
| FmsDashboard quota table — no drugs with quota | Empty | Existing "No controlled drug quotas set" pattern; add Pesara column with `"-"` |
