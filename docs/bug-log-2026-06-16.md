## BUG-001 · CRITICAL · ProtectedRoute: `/specialist` exposes wrong roles

**File:** `src/components/ProtectedRoute.tsx:15`

**Status:** ✅ Fixed

**Description:**
The `/specialist` route (SpecialistDashboard) was mapped to `["admin", "pharmacist"]`. This is doubly wrong:
- `fms` users could not access their own dashboard at `/specialist`
- `pharmacist` users could access the specialist approval UI they have no business using

Per CLAUDE.md the intended access is `fms` only (plus `admin`).

**Before:**
```ts
{ prefix: "/specialist", roles: ["admin", "pharmacist"] },
```

**After:**
```ts
{ prefix: "/specialist", roles: ["admin", "fms"] },
```

## BUG-002 · HIGH · DrugMaster: `isAdmin` checks pharmacist role instead of admin

**File:** `src/pages/DrugMaster.tsx:62`

**Status:** ✅ Fixed

**Description:**
`isAdmin` was derived as `role === "pharmacist"`, which inverts the intended logic:
- `admin` users got `isAdmin = false` → lost access to edit and opening-balance controls
- `pharmacist` users got `isAdmin = true` → incorrectly gained those controls

Both roles legitimately access DrugMaster (the route allows both), so the fix grants `isAdmin` to either.

**Before:**
```ts
const isAdmin = role === "pharmacist";
```

**After:**
```ts
const isAdmin = role === "admin" || role === "pharmacist";
```

## BUG-003 · HIGH · Quota counting uses `"approved"` status that never exists on `dispensing_requests`

**Files:**
- `src/pages/FmsDashboard.tsx:273`
- `src/pages/SpecialistDashboard.tsx:101`

**Status:** ✅ Fixed

**Description:**
Both dashboards count annual drug quota usage by filtering `dispensing_requests` where `status IN ("approved", "fulfilled")`. However, FMS approval always sets status to `"pending_pharmacy"` (not `"approved"` — that string is only used for `antibiotic_forms`). This means all requests that have been approved but not yet dispensed are invisible to the quota counter, causing the displayed quota consumption to be understated.

**Before (both files):**
```ts
.in("status", ["approved", "fulfilled"])
```

**After (both files):**
```ts
.in("status", ["pending_pharmacy", "fulfilled"])
```
